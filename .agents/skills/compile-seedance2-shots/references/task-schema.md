# Seedance 2.0逐镜任务结构

## 项目层

- `project_id`
- `model_profile`
- `source_lock`
- `production_assumptions`
- `bibles`
- `assets`
- `asset_generation_tasks`
- `shots`

只允许使用 `shots` 作为任务数组名称，禁止同时出现 `tasks`、`clips` 等平行格式。

## 每镜必填

- `shot_id`：唯一编号。
- `source_ids`：对应原文或对白编号。
- `purpose`：该镜推进的信息或情绪。
- `duration_seconds`：4至15秒。
- `aspect_ratio`：例如9:16。
- `input_mode`：text、image、audio、video或mixed。
- `references`：资产ID、类型和用途。
- `keyframes`：首帧图片提示词、验收标准，以及按需提供的尾帧图片提示词。
- `first_frame`：人数、位置、朝向、视线、手部、道具、背景和光线。
- `camera`：景别、机位、角度、镜头感、构图和单一主要运镜。
- `blocking`：人物和道具如何移动。
- `performance`：可观察的表情、呼吸、姿态和动作。
- `audio`：说话者、逐字台词、对象、声音、节奏、停顿、口型和必要剧情内声音。默认不生成背景音乐、配乐、歌词、背景声铺底或环境声铺底；必须保留人物说话声音和场景中真实发生且应当发出的事件声；后期发布时添加字幕、背景音乐和声音包装。
- `voice_continuity`：角色音色ID、音色描述、参考音频路径、跨镜继承要求、禁止变化。
- `timeline`：逐秒可执行动作。
- `edit_handles`：后期剪辑手柄，包含开头稳定时长、结尾稳定时长、可裁切状态和接剪说明。
- `lighting_environment`：光影和环境变化。
- `end_frame`：最终姿势、视线、道具、构图和保持时间。
- `continuity_from`：继承上一镜的状态。
- `negative_constraints`：本镜真正相关的禁止项。
- `prompt`：可直接复制的中文提示词。
- `acceptance`：可以判断通过或失败的标准。
- `fallback`：失败时先删减什么。

## 提示词顺序

```text
【任务与时长】
【参考资产】
【关键帧引用】
【角色锁定】
【场景与时间】
【首帧】
【摄影】
【人物走位与动作闭环】
【表演与情绪】
【音画同步】
【角色音色】
【逐秒时间轴】
【剪辑手柄】
【光影、风格与环境】
【尾帧】
【连续性】
【禁止】
```

不要使用只有“电影感、温馨、高级”等抽象词的提示。必须把抽象风格翻译成可见的光线、距离、色彩、动作和摄影选择。

## 单镜头骨架约束

- 角色锁定必须写清年龄、性别、外貌、服装、身份、固定道具或固定动作锚点、声音特征；跨镜角色必须说明保持不变的部分。
- 场景与时间必须写清地点、年代/成长阶段、日夜或具体时段、天气/季节、主要光源和空间锚点。
- 摄影必须只保留一个主要景别、一个机位和一个运镜；禁止同镜快速切换、多角度、多运镜。
- 动作必须有闭环：起始状态、执行过程、对象或道具去向、完成后的最终可见状态、尾帧保持状态。
- 音频必须写清说话者、对象、逐字台词、动作完成后才开口的触发条件、语气、音色、语速、停顿、口型和非说话者嘴部状态。
- 关系戏必须写清人物距离、身体朝向和视线目标；有对白时必须说明说话者看向谁，不得让角色对空气或对镜头说本应说给他人的话。
- 可移动道具必须写清所有权变化：谁拿着、递向谁、谁接稳、谁松手、最终停在哪里。
- 手机、便签、包装等文字必须写清观看对象：观众读字、角色阅读或角色对手机语音，三者不能混写。
- 对白必须使用白名单约束，说明不得新增、重复、改写或提前说出白名单外台词。
- 情绪节奏必须落到可见画面和声音变化，不能只写“压抑、克制、神圣、爆款感”。整条视频的前3秒钩子、中段递进、结尾反转/升华/留白，应拆解到对应镜头的逐秒时间轴。

## 独立执行要求

- 每个 `prompt` 必须能够脱离其他镜头文本独立使用。
- 每个 `prompt` 必须把剪辑手柄写成生成要求：开头稳定、结尾稳定、最后稳定段不新增关键动作或对白。
- 禁止使用“同上”“沿用上一镜设置”“人物不变”“场景不变”“略”等省略表达。
- 引用前镜时必须写成具体状态，例如人物位置、朝向、服装、手中道具和情绪。
- 未提供的资产不得写成已经存在；应放入 `asset_generation_tasks`，并在镜头中标注“生成并验收后引用”。
- 精确文字、字幕、手机界面和包装内容如果计划后期完成，必须明确写入验收和禁止项，不能要求视频模型凭空稳定生成。
- 每条分镜 prompt 默认必须写明“不生成背景音乐、不生成背景声铺底、不生成歌词、不生成字幕；后期再添加字幕、背景音乐和声音包装”。如果没有必要对白，应写“无对白，保留必要剧情内声音，不生成背景音乐，后期添加声音包装”。

## 交付一致性

人类可读逐镜稿与 JSON 任务包必须一致：

- 镜头 ID 一致。
- 台词逐字一致。
- 时长一致。
- 资产 ID 一致。
- 高风险等级和验收标准一致。
- 剪辑手柄时长、尾帧保持和接剪说明一致。
