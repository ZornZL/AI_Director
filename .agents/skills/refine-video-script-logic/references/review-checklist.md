# 视频文案逻辑检查表

## 1. 基础成立

- 场景、人物和关系是否一眼可懂。
- 每个人当前想得到什么、害怕什么或误解什么。
- 人物行为是否符合其已知信息和处境。
- 道具、金钱、手机、车辆和产品是否有清楚用途。
- 是否存在前一句和后一句互相冲突。

## 2. 钩子与信息

- 前3秒是否出现问题、反差、冲突或结果悬念。
- 钩子是否与后文真正相关，而不是标题欺骗。
- 每一段是否增加新信息。
- 观众是否在正确时间知道正确的信息。

## 3. 反转

- 反转前是否建立了一个明确但错误的判断。
- 是否有公平线索，但没有提前说破。
- 揭晓信息能否重新解释前面的行为。
- 反转是人物和因果产生，还是突然添加新设定。
- 揭晓后是否还有角色反应或第二笑点。

## 4. 对白

- 每句话是否符合人物身份、情绪和关系。
- 是否存在讲给观众听而不是人物会说的话。
- 能否删除重复信息而不损失情节。
- 长句是否需要在自然语义停顿处拆开。
- 重要笑点或情绪前后是否留有反应时间。

## 5. 产品轻植入

- 产品为什么在这个场景出现。
- 产品参与的是动作、关系还是情节因果。
- 产品和金额、身份、行为之间是否造成误解。
- 是否直接或间接承诺健康功效。
- 是否能用独立低运动特写准确展示包装。
- 产品删除后故事是否仍成立；加入后是否更顺畅。

## 6. 品牌与合规

- 不低俗、不羞辱、不制造群体对立。
- 不涉及国家政治立场、宗教宣传或迷信判断。
- 不使用疾病、治疗、保证性效果和身体焦虑推动产品。
- 不复制参考作品的完整故事、台词、人物或标志性画面。
- 真人、车辆品牌、音乐、Logo和第三方资产是否需要授权或替换。

## 7. Seedance脚本风险

- 是否多人同框连续轮流说话。
- 是否同一段同时包含说话、复杂动作、运镜和产品展示。
- 是否依赖准确手机界面、金额、字幕或包装文字。
- 是否有车辆、液体、开封、递交或精细手部动作。
- 是否存在多场景、换装、时间跳跃和大量角色。

脚本初审只标记这些风险。具体拆镜和提示词由Seedance编译器完成。
