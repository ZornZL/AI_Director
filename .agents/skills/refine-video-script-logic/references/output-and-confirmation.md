# 输出与用户确认格式

## 1. 原稿核心锁定

- 核心故事：
- 人物关系：
- 核心反转：
- 结尾：
- 必须保留：
- 用户允许调整：

## 2. 总体判断

用一段话说明文案是否成立、最有价值的机制和最大问题。不要只给分数。

## 3. 问题清单

| 等级 | 原文位置 | 问题 | 为什么 | 不修改的影响 |
|---|---|---|---|---|

## 4. 最小修改建议

| 编号 | 原文 | 建议 | 解决的问题 | 是否改变原意 |
|---|---|---|---|---|

## 5. 候选版本

- A版：原稿保留版，只加风险备注。
- B版：逻辑微调版，只处理P0、P1和用户同意的P2。
- C版：生成友好版，压缩重复对白或简化高风险动作，不改变人物关系、核心梗和结局。

没有必要时不要输出C版。

## 6. 变更对照

明确列出删除、增加、改写和位置调整，避免用户逐字比对全文。

## 7. 用户确认提示词

```text
请确认本次文案采用哪种处理方式：

A：保留原稿，不修改；风险留到分镜阶段处理。
B：采用逻辑微调版。
C：采用生成友好版。
D：组合选择，请写明采用或拒绝的修改编号，例如“采用1、3，拒绝2”。
E：暂不锁定，继续讨论。

你也可以补充：
- 必须原样保留的台词：
- 绝对不能改变的情节：
- 可以压缩的内容：
- 希望加强的方向：搞笑 / 反转 / 温情 / 哲理 / 产品自然度 / 生成稳定性

在你明确确认前，我不会把任何候选修改当作正式文案，也不会进入Seedance 2.0逐镜编译。
```

## 8. 确认后的锁定格式

```text
文案状态：已锁定
采用版本：
确认日期：
核心保留：
已采用修改：
未采用建议：
下一步：交给 compile-seedance2-shots
```
