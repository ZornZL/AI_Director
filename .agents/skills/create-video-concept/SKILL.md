# 视频创意策划

## 前置读取

只读取品牌定位、目标用户、本次产品卡、审美偏好和最相关的少量结构卡。

## 工作流程

1. 明确传播目标、平台、片长、预算和合规边界。
2. 如果有活动规则、参考视频或爆款样片，先提取可学习的结构机制、情绪曲线、钩子、反转、结尾互动和生成风险；同时列出禁止照搬的人物、台词、完整情节链和具体桥段。
3. 从产品卖点、用户命题或参考结构中寻找现实冲突、人物愿望和情绪价值。必须把参考机制迁移成新人物、新情境、新因果，不能只替换表层名词。
4. 先进入 G0爆款方向门，对用户给出的主题或命题生成 3-5 个故事方向，不得直接收敛成单一脚本。每个方向必须说明一句话命题、前3-8秒钩子、观众停留理由、人物关系、核心冲突、3-5层情绪阶梯、最重的一句话、结尾回扣、产品或品牌进入方式、积极价值和生成难度。
5. 对 3-5 个方向进行二次筛选，使用评分表比较命题贴合度、原创性、停留钩子强度、前8秒信息密度、情绪递进强度、台词含蓄度、关系代价、结尾回扣力度、评论区讨论潜力、产品/品牌自然度、Seedance可制作性、合规风险、完播风险和近期题材重复惩罚。
6. 对推荐方向做8秒开场测试。至少给出认知错位型、轻问题重答案型、反常动作/关系悬念型三种开场，比较停留理由、信息密度、可拍性和风险；选出最强开场作为后续脚本起点。
7. 选出 1 个推荐方向，并说明为什么它优于其他方向；同时保留 1 个备选方向，避免过早锁死。
8. 用户确认方向后，才发展成完整视频脚本；如果用户明确要求“直接给脚本”，也应先用简表展示 3 个候选方向、G0判断、8秒开场测试和筛选理由，并要求用户锁定方向。未确认前最多输出推荐方向的脚本骨架或前8秒开场，不输出完整正片脚本。
9. 完整脚本必须说明钩子、人物关系、冲突、情绪递进、反转/高潮、结尾互动、产品进入、关键台词的说话对象与未说出口的代价，以及不可复制参考片的边界。
10. 输出后必须把脚本交给 `refine-video-script-logic` 做逻辑初审和 A/B/C/D/E 确认；未经用户锁定，不进入导演分镜、资产或 Seedance 逐镜任务。

用户选择方向前，不得直接制作完整分镜或批量生成素材。

## G0爆款方向门

G0用于判断一个方向值不值得写成完整文案。每个候选方向必须填完以下内容：

```text
一句话命题：
前3-8秒钩子：
第8秒停留理由：
第20秒继续观看理由：
人物关系：
核心冲突：
情绪阶梯：
最重的一句话：
结尾回扣：
产品/品牌进入：
Seedance生产风险：
爆款风险：
```

判断标准：

1. 前3-8秒不能只是标题或旁白，必须有认知错位、轻问题重答案、反常动作或关系悬念。
2. 中间不能是并列素材，必须至少有3层情绪递进。
3. 角色不能只有身份标签，必须有处境、关系对象和现实压力。
4. 最重的一句话必须短、具体、有对象、有未说出口的代价；不得是“人生就是……”这类口号。
5. 结尾必须回扣开头同一动作、物件、台词、问题或关系。
6. 如果一个方向只能说明主题，不能说明停留、递进、关系代价和回扣，必须降权或淘汰。

## 8秒开场测试

推荐方向进入完整脚本前，必须优先测试三个开场：

1. 认知错位型：人物身份、关系或动作不合常规但真实。
2. 轻问题重答案型：问题像玩笑，答案逐步变重。
3. 反常动作/关系悬念型：角色在不该等、不该忘、不该说的位置制造悬念。

每个开场必须包含：

- 前3秒画面或第一句话。
- 第3-8秒释放的新信息。
- 观众为什么停。
- Seedance生成风险。

若用户指定开场，也要用这三类做对照，说明沿用或替换的理由。

## 爆款叙事前置要求

当选题属于命题短片、情绪短片、参考爆款迁移、群像答案型文案、亲情、友情、遗憾、成长或错过主题时，候选方向必须额外说明四件事：

1. 前 3-8 秒用什么让观众停留。
2. 中间如何从轻到重递进，至少列出 3 个情绪台阶。
3. 每个主要段落的最小故事单元是什么，而不是只给金句或答案。
4. 结尾回扣开头的哪个动作、物件、台词、问题或关系。
5. 关键台词如何做到精简但有余味：每句说给谁、没说出口的代价是什么、哲理如何藏在关系里。

如果一个方向只能给主题和金句，不能说明停留、递进、故事和呼应，必须降权，不得作为默认推荐。

## 答案型文案说话关系表

如果方向采用街采、群像回答、多人依次回答同一问题、答案池或金句型结构，在生成完整脚本前必须先给候选答案建立说话关系表：

```text
角色身份 → 说话对象 → 缺席关系 → 未说出口的代价 → 画面锚点 → 人生哲理
```

要求：

1. 角色身份必须包含年龄、职业/处境、家庭或关系压力。
2. 说话对象不能默认是观众；必须找出真正说给谁。
3. 缺席关系可以是亲人不在场、朋友离开、过去的自己、逝去的人，也可以是面对面却说不出口。
4. 每个答案必须说明没说出口的代价。
5. 每个答案必须有可见画面锚点。
6. 哲理用于创作判断，不直接写成人物台词。
7. 入片答案必须短。优先使用一口气能说完的短句、半句话、停顿后的补句或具体动作后的低声表达；不得用长台词解释人生道理。

未完成该表时，不输出正片文案。

## 角色台词精简深意规则

所有重要角色台词必须满足：

1. 短：一句话尽量只表达一个动作、一个对象或一个没说出口的情绪。
2. 准：必须知道说给谁，不默认说给观众。
3. 藏：人生哲理藏在关系、代价、物件和停顿里，不直接说破。
4. 重：这句话不说会有后果，或者已经来不及补说。
5. 可演：画面中有动作、眼神、物件或空间能承接这句话。

优先写法：

```text
“放到挂电话前吧。我想说，我也想你。”
“少掉关门前那句，别管我。”
“她每次都等我先挂。那天也是。”
```

禁用写法：

```text
“人生最大的遗憾，就是没有珍惜眼前人。”
“亲情是世界上最伟大的爱。”
“我们都应该学会表达爱。”
```

## 段落深挖规则

当用户指出某段“不够有画面”“不够深刻”“想表达什么”时，不直接堆更重的金句。先回答该段在全片中的功能，再用可见物件、动作、空间、停顿和可听见的原台词改写。改写必须能让观众不读解释也理解人物的遗憾、选择或反转，并说明新版本比原版本强在哪里。

## 题材去重与惯性校正

当近期项目或参考分析已经使用过相近人物关系、核心困境、情绪钩子或产品进入方式时，必须主动降权重复方向。特别是已经连续出现过“妈妈/母女/亲情照护/失忆或记忆退化/离世遗物/物件回扣”时，除非用户明确点名，否则不得把它作为默认推荐方案。

遇到抽象命题（如遗忘、末日、不在场、后来、答案等）时，先做多义展开，再选方向。至少列出五种不同解释路径，并覆盖不同关系类型，例如职业、友情、社区、陌生人互助、职场、城市公共事件、个人成长或老板IP观点；不得只把命题自动解释成父母子女亲情。

如果用户只要求出现品牌名（例如“如是心”三个字）而没有指定具体产品，不得自动调用某个产品卡里的高情绪亲情母题。优先考虑招牌、便签、公益墙、店名、包装远景、活动名、文件夹名等低侵入出现方式。

三个原创方向必须在以下维度中至少三项互不相同：人物关系、主冲突、命题解释、产品/品牌进入方式、反转机制、结尾动作。评分时增加“重复惩罚”：与最近已做项目相似的人物关系、疾病/遗忘机制、遗物证据、亲情信物或结尾回扣，每命中一项都要扣原创性分，并说明替代方案。
