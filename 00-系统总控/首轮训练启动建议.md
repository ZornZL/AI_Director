# 首轮训练启动建议

## 第二步：确定首轮训练目标

### 推荐目标

用原创虚拟人物制作一条20至30秒的双人反转正能量剧情，重点验证：

1. 简单文案能否稳定编译成Seedance 2.0逐镜任务。
2. 两个虚拟人物能否跨镜保持身份和服装一致。
3. 双人对白能否通过单人镜头和正反打控制说话者与口型。
4. 首帧、尾帧和背景锚点能否维持空间连续性。
5. 失败后采用拆镜、固定机位和减少动作是否有效。

### 为什么首轮不建议老板真人出镜

- 老板真人具有更高的肖像授权、身份一致性和发布风险。
- 当前首先要验证编译流程和Seedance生成规律，不应同时承担老板形象试错。
- 虚拟人物项目跑通后，再把已经验证的镜头规则用于老板IP内容。

### 为什么首轮不建议植入产品

- 产品包装、文字、手部和人物表演会同时增加生成难度。
- 先验证人物、对白和连续性，第二或第三个项目再训练产品轻植入，容易判断失败原因。

### 建议的视频规格

- 平台母版：抖音。
- 画幅：9:16。
- 总时长：20至30秒。
- 人物：2名原创虚拟人物。
- 场景：1个固定生活场景。
- 镜头数：4至6个。
- 台词：每人1至2句，单句简短。
- 摄影：固定或轻微运动，以中景、近景和反应镜头为主。
- 结构：冲突或误解开场 → 信息递进 → 善意或价值反转 → 留有余味的结尾。
- 产品：首轮不出现。

### 推荐参考视频数量

选择3条参考视频，每条只训练一种主要能力：

1. 一条学习前3秒冲突或悬念。
2. 一条学习双人正反打、反应镜头和情绪停顿。
3. 一条学习结尾反转、哲理回扣或余味。

不得把三条参考片的完整故事拼接在一起，只迁移经过用户确认的镜头和叙事机制。

### 首轮完成标准

- 形成一份完整Seedance 2.0逐镜任务包。
- 角色正脸、最复杂动作和对白镜头三个测试全部通过。
- 至少连续3个镜头的人物、服装、空间和道具状态基本一致。
- 每次失败都有记录，并且每次只修改一到两个变量。
- 最终形成至少1条有效规则和1条失败避坑经验。

## 第三步：建立首个正式项目

### 建议项目

- 项目编号：`ABD-2026-001-virtual-dialogue-reversal`
- 中文名称：虚拟人物双人反转正能量短片
- 当前状态：候选，等待简单文案确认后创建项目目录
- 核心目标：验证Seedance 2.0逐镜编译、虚拟角色一致性、对白正反打和连续性。
- 非目标：老板真人IP、产品植入、复杂运镜、多场景和大动作。

### 建议的简单文案结构

```text
环境：一个固定、容易理解的日常生活场景。
人物：两名关系明确的原创虚拟人物。
开头：其中一人做出容易被误解的行为。
对白：另一人提出质疑或冲突。
递进：通过一个动作、物品或简短台词提供新信息。
反转：观众发现最初的判断是错误的，真实动机是善意或责任。
结尾：用开头出现过的物品或动作完成回扣，不喊口号。
```

### 项目启动前需要用户确认

1. 更偏搞笑反转、温情反转还是哲理反转。
2. 两名人物的关系和大致年龄。
3. 一个固定场景。
4. 必须保留的对白或结局。
5. 是否提供参考视频；如果提供，每条最想学习什么。

### 后续项目顺序

- 项目2：虚拟人物加单个产品轻植入，重点训练产品因果和低运动特写。
- 项目3：老板IP观点或故事内容，使用已验证镜头语言；真人形象必须走绑定设备与授权流程。
- 项目4：加入动作参考、匹配转场或复杂情绪变化。
- 项目5：根据抖音、小红书、快手和视频号数据进行平台版本适配。
