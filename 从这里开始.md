# AiBianDao 从这里开始

这是一个以 Seedance 2.0 为主要生产模型的“专属 AI 视频编导”系统。当前先稳定提供一个核心能力：把用户的简单视频文案编译成由独立分镜组成、可逐镜投喂 Seedance 2.0 的完整生产脚本。

> **全局制作方式：本系统内的视频全部使用 Seedance 2.0 生成，不采用现实拍摄。** 文档中的摄影、机位、灯光、表演、场景、演员、道具和调度，均指对 AI 生成画面的模拟控制，不代表组织真人演员、摄影团队或现实场地拍摄。

系统有两个入口，必须先判断本次属于哪一种：

```text
创意入口：主题/命题/参考/产品故事 → G0爆款方向门 → 3-5个候选方向
→ 8秒开场测试 → 用户锁定方向 → 完整文案草案
→ 文案逻辑初审与A/B/C/D/E确认 → 唯一锁定文案
→ 发布平台/画幅/片长确认 → 导演脚本与逐镜拆分 → 资产与关键帧要求 → Seedance 2.0逐镜任务包

生产入口：用户已有具体简单文案/剧情原稿 → 文案逻辑初审与用户确认 → 唯一锁定文案
→ 发布平台/画幅/片长确认 → 导演脚本与逐镜拆分 → 资产与关键帧要求 → Seedance 2.0逐镜任务包
→ 用户手动生成与验收
```

如果用户说“帮我出视频文案、写一个脚本、根据这个主题/产品/参考片做创意”，默认走创意入口，先给方向，不直接给唯一完整文案。只有当用户已经提供可识别的具体文案或剧情原稿，并要求审稿、润色、分镜或生产时，才走生产入口。

G0爆款方向门必须先判断：前3-8秒怎么让人停、中间情绪如何递进、角色每句关键台词说给谁、未说出口的代价是什么、结尾如何回扣开头并深化主题。角色说话必须精简，但要让观众能从关系和代价里读出人生哲理，不得把哲理直接写成口号。

进入导演分镜前必须确认发布平台、画幅比例、目标片长和当前 Seedance 入口参数。用户未确认时，所有脚本、分镜、资产计划和逐镜任务包只能写“待确认”或列出候选，不得默认 9:16、16:9、抖音、横屏或竖屏。

## 第一次使用

1. 使用 `04-模板库/Seedance2简单文案输入模板.md` 提交已知内容；不知道的字段可以留空。
2. 有品牌或产品时读取对应资料；纯剧情项目可以跳过产品资料。
3. 调用 `refine-video-script-logic` 检查因果、反转、对白和生成风险，等待用户选择 A/B/C/D/E。
4. 用户选择 A/B/C/D 并形成唯一锁定文案后，先确认发布平台、画幅比例、目标片长和当前 Seedance 入口参数；未确认时不得写死画幅。
5. 调用 `compile-seedance2-shots`。
6. 输出完整导演脚本、逐镜任务、资产与关键帧要求、连续性、验收和失败降级方案。
7. 逐镜网页端提示词默认采用已验证的 Seedance 执行结构：硬约束前置、本镜核心目标、资产/角色锁定、单一镜头控制、声音白名单、逐秒时间线、稳定尾帧、禁止项收束。
8. 用户在即梦或当前 Seedance 入口逐镜生成；先测试身份、最难动作和产品/文字等高风险镜头。
9. 生成自动化、剪辑和发布复盘属于后续扩展，不影响当前逐镜脚本能力使用。

详细边界见 `00-系统总控/当前系统能力边界.md`。

## 参考视频默认新手模式

用户不需要预先判断视频里哪些内容值得学习。每次收到参考视频后，系统默认：

1. 基于真实画面主动完成结构、镜头、转场、光影、声音、表演和产品植入分析。
2. 将发现分为：`建议学习`、`谨慎学习`、`不建议学习`、`禁止照搬`。
3. 说明判断依据、适用场景、生成难度、迁移方法和与当前能力的差异。
4. 自动提出候选训练点和下一步验证建议，不要求用户掌握专业术语。
5. 分析和候选优化可以自动进行；正式正向入库仍需用户明确确认。
6. 入库后继续通过原创迁移、生成复盘和发布数据修正规则，分别升级到L2和L3。

## 新对话框统一开场

```text
这是 AiBianDao 项目的专职任务。
请先读取《从这里开始.md》、总索引和当前能力地图，
再按本次任务调用相关 Skill。只读取任务需要的资料。
```

## 四类对话框

- 总控：检查项目状态、能力缺口和训练方向。
- 参考视频分析：一条视频一个，沉淀后归档。
- 视频创作：当前核心入口，一个正式视频项目一个，完成简单文案到 Seedance 逐镜任务包。
- 数据复盘：一条已发布视频一个，复盘后归档。

## 三条底线

1. 归档对话框不等于知识入库，关键结论必须写入文件。
2. 未经用户确认的分析只能作为候选，不能成为稳定规则。
3. 单条视频的过程文件只放在自己的项目目录，避免多个“最终版”。
4. “可直接执行”不等于“一次必成”；每镜必须有验收和失败降级方案。
