# 工作流程

> 本流程是 Seedance 2.0 AI 视频生成流程，不是现实拍摄流程。所有镜头设计最终都必须转译为模型可执行的生成任务。

## 当前核心生产流程

```text
简单文案已提交 → 逻辑初审待确认 → 文案已锁定 → 导演与分镜待审
→ 资产和关键帧要求待审 → Seedance任务包待审 → 可交付
```

`可交付`表示每个镜头已经形成可独立复制到 Seedance 2.0 的执行任务，并且已经给出核心资产、逐镜网页端参数、验收标准和失败降级方案。当前系统到此完成稳定交付；后续生成由用户手动完成，系统可以继续提供抽帧复盘、音画基础检测、失败记录和单变量重试建议。

## 脚本创意到分镜强制链路

每次进行脚本创意、选题迁移、参考视频改编、命题故事、产品故事或剧情软广时，必须按以下链路推进。不得从一个灵感直接跳到完整分镜或 Seedance 任务包。

```text
平台/规则/参考输入确认 → 参考结构拆解与禁抄边界 → G0爆款方向门
→ 多方向候选 → 8秒开场测试 → 用户锁定方向
→ 完整文案草案 → 逻辑初审与A/B/C/D/E确认 → 弱段画面化深挖 → 唯一锁定文案
→ 发布平台/画幅/片长/入口参数确认 → 导演分镜 → 资产/声音/关键帧计划 → Seedance逐镜任务包
```

执行要求：

1. 如果用户提供活动规则、参考视频或优秀案例，先提取可学习的结构机制、情绪曲线、钩子、反转、结尾互动和生成风险；同时明确人物、台词、完整情节链和具体桥段不得照搬。
2. 创意阶段先给 3-5 个差异明显的候选方向，至少比较命题贴合度、原创性、停留钩子强度、前8秒信息密度、情绪递进强度、台词含蓄度、关系代价、结尾回扣力度、评论区讨论潜力、Seedance可制作性、合规风险和近期题材重复惩罚；用户确认方向前，不得直接制作完整分镜或批量素材。
3. 每个候选方向必须通过 G0爆款方向门：一句话命题、前3-8秒钩子、观众继续观看理由、人物关系、核心冲突、3-5层情绪阶梯、最重的一句话、结尾回扣动作/物件/台词、Seedance难度和爆款风险。不能说明这些内容的方向不得作为默认推荐。
4. 对抖音情绪短片、命题短片、参考爆款迁移、答案型/群像文案、产品故事、亲情/友情/遗憾/人生哲理类短片，完整文案前必须做8秒开场测试。至少比较认知错位型、轻问题重答案型、反常动作/关系悬念型三种开场；选出最强开场或说明为何沿用用户指定开场。
5. 用户锁定方向后，先形成完整视频文案，再进入 `refine-video-script-logic`。未经用户明确选择 A/B/C/D/E，不得把候选修改当作正式文案，也不得进入 `compile-seedance2-shots`。
6. 逻辑微调时，优先保留核心命题、人物关系、情绪递进和用户指定台词；只修复动机、因果、信息释放、反转成立性、节奏和可生成风险。
7. 角色关键台词必须精简、关系明确、含义有余味。每句重要台词都要能回答：这句话说给谁、为什么此刻难说、没说出口的代价是什么、画面里由什么物件/动作承载、隐含的人生哲理是什么。哲理只能隐含在关系和代价中，不得写成角色口号。
8. 对用户指出“不够有画面、不够深刻”的段落，必须先说明该段在全片承担的功能，再用可见物件、动作、空间和停顿替代抽象解释。判断标准是观众不读旁白也能从画面理解遗憾、选择或反转。
9. 每次段落改写都要给出可替换文本，并说明它比原段强在哪里；用户确认后才同步到项目脚本、分镜、资产清单、声音规则和逐镜操作单。
10. 进入逐镜任务包后，继续遵守“分镜脚本修改同步门槛”：任何新增、删改、拆分、合并或顺延都必须同步受影响文件和镜头编号。

## 发布平台与画幅确认门

G1 文案锁定后、G2 导演分镜前，必须先确认发布平台、画幅比例、目标片长和当前 Seedance 入口参数。画幅会影响构图、角色站位、字幕安全区、首帧比例、关键帧设计、镜头合并策略和网页端参数，不能由系统默认写死。

执行要求：

1. 用户已经明确给出发布平台和画幅时，按用户确认值写入项目状态、分镜和任务包。
2. 用户只给出平台但未给出画幅时，只能列出候选画幅和推荐理由，并等待用户确认；不得把抖音自动等同于 9:16，也不得把视频默认等同于竖屏。
3. 用户未确认平台或画幅时，所有脚本、导演分镜、资产计划、关键帧计划和 Seedance 逐镜任务包必须写“发布平台待确认 / 画幅待确认”，不能写成 9:16、16:9、横屏或竖屏。
4. 若为了讨论构图需要临时假设画幅，必须标注为“临时构图假设，不可作为生产参数”，并在进入 Seedance 任务包前再次确认。
5. 一旦画幅确认，必须同步修改项目状态、当前生效分镜、资产生产清单、首帧/尾帧比例、逐镜任务包、项目看板和所有网页端操作参数。

## G0爆款方向门

G0用于阻止普通完整文案过早成稿。凡是系统从主题、产品、命题、参考或空白需求开始创作文案，必须先过G0，再进入完整文案。

每个候选方向必须包含：

1. 一句话命题：这条视频到底讨论什么。
2. 前3-8秒钩子：认知错位、轻问题重答案、反常动作或关系悬念。
3. 用户停留理由：第8秒、第20秒分别有什么新信息让观众不划走。
4. 人物关系：不是“一个人/一个老人/一个女孩”，而是带处境、压力和关系对象的人。
5. 核心冲突：人物想要什么，为什么此刻难。
6. 情绪阶梯：至少3层，建议按“轻冲突 → 现实处境 → 关系代价 → 不可逆遗憾 → 主题回扣”递进。
7. 台词潜力：最重的一句话必须短、具体、有对象、有未说出口的代价，不能是抽象格言。
8. 结尾回扣：回到开头的同一动作、物件、台词、问题或关系，并让它变得更重。
9. 生产风险：Seedance中最难生成的身份、动作、文字、产品或声音风险。

评分时必须额外比较：停留钩子强度、前8秒信息密度、情绪递进强度、台词含蓄度、关系代价、结尾回扣力度、评论区讨论潜力、完播风险。

未通过G0的方向，只能作为废案或待重构方向，不能进入完整文案。

## 爆款叙事验收门

命题短片、情绪短片、参考爆款迁移、群像答案型文案和亲情/友情/遗憾类项目，在进入导演分镜前必须额外通过“停留-递进-故事-呼应”验收。该规则用于防止文案变成散文化金句或并列答案合集。

验收问题：

1. 前 3-8 秒是否有明确停留钩子：认知错位、轻问题重答案、异常动作或关系悬念。
2. 前 20 秒是否给出继续观看理由，而不是只完成主题介绍。
3. 中段是否至少有 3 次情绪升级，且案例顺序不能随意调换。
4. 每个案例是否有最小故事单元：角色是谁、此刻想要什么、为什么难、选择或没选择什么、留下什么余味。
5. 最后三个段落是否明显重于前三个段落。
6. 结尾是否回扣开头同一动作、台词、物件、问题或关系。
7. 主题深化是否来自故事结果，而不是作者旁白直接宣布。
8. 关键台词是否精简，有具体说话对象和未说出口的代价；是否让观众自己悟到哲理，而不是角色直接讲道理。

未通过时，不得进入 `build-video-script-storyboard` 或 `compile-seedance2-shots`；必须退回文案阶段，先补开头停留方案、情绪递进阶梯、角色故事单元和首尾回扣动作。

适用训练卡：

- `02-编导训练库/结构库/ABD-STRUCT-2026-005-停留递进呼应爆款情绪结构_v01_L1.md`
- `02-编导训练库/失败与避坑库/ABD-RISK-2026-003-散文化文案与答案合集风险_v01.md`

## 答案型文案说话关系表

街采式、群像式、命题问答式、多个角色依次回答同一问题的文案，在进入 G2 正片文案前必须先完成“说话关系表”。该规则用于避免角色只对观众讲金句，而没有具体对象、关系和代价。

强制链路：

```text
角色身份 → 说话对象 → 缺席关系 → 未说出口的代价 → 画面锚点 → 人生哲理
```

执行要求：

1. 每个候选答案必须说明角色身份，不能只写年龄或职业标签。
2. 每句答案必须说明真正说话对象：采访者、在场人物、缺席亲人、过去的自己、逝去的人或某段关系。
3. 必须说明关键关系是否缺席；若不缺席，也要说明为什么面对面仍说不出口。
4. 必须说明这句话没说、没做或没等会留下什么代价。
5. 必须给出一个可见画面锚点承载情绪。
6. 人生哲理只作为创作判断，不得直接塞进人物台词。
7. 入片台词优先短句、半句话、停顿、反问或具体动作后的低声表达；能用物件和动作表达的，不用长台词解释。

未完成说话关系表时，不得进入正片文案、导演分镜或 Seedance 逐镜任务包。

适用训练卡：

- `02-编导训练库/结构库/ABD-STRUCT-2026-006-答案型文案说话关系表_v01_L1.md`

## 当前六个人工确认门

1. G0：爆款方向、8秒开场、情绪阶梯和结尾回扣确认。
2. G1：原始文案逻辑调整与唯一锁定确认。
3. G1.5：发布平台、画幅比例、目标片长和当前 Seedance 入口参数确认。
4. G2：导演假设、分镜、角色、场景和产品外观确认。
5. G3：资产、关键帧、连续性和高风险测试方案确认。
6. G4：逐镜 Seedance 任务包交付确认。

审批结果只能是：`通过`、`修改后再审`、`终止`。

## 角色声音与无BGM无字幕确认门

G1 文案锁定后、G2 导演分镜前或分镜初期，凡锁定文案包含人物对白、手机语音、旁白或关键剧情内声音，必须先补齐 `../04-模板库/角色声音与无BGM无字幕规则确认模板.md` 中的声音口径。该确认属于当前生产流程的规则/模板沉淀，不新建 Skill。

确认内容必须包括：

1. 旁白是否由 Seedance / 即梦模型直接生成；若用户确认模型直出，旁白必须作为 `VO_NARRATOR` 写入角色声音圣经和逐镜提示词。
2. 每个说话角色的声音ID、年龄感、音色、语速、情绪底色、禁止项和台词白名单。
3. 每个镜头允许保留的剧情内声音，例如人物对白、手机语音、旁白、脚步、门响、碗筷、锅盖、水龙头、手机震动和电梯提示音。
4. Seedance 阶段默认不生成字幕、片名、片尾字卡、屏幕文案、水印、BGM、歌词、背景声铺底或环境音乐；字幕、片尾文案、配乐、环境氛围和最终混音在后期添加。
5. 手动逐镜生成时，每个分镜视频脚本和可复制提示词必须自包含声音规则：角色音色、说话对象、台词白名单、触发条件、非说话者嘴部状态、剧情内声音和禁止项，不能只引用独立声音圣经。

## 核心资产硬门槛

在建议、提交或登记任何正式视频生成前，必须先完成核心资产检查。若项目仍缺少主角、产品、核心场景、关键文字道具或高风险镜头首帧中的任一高优先资产，系统不得把下一步表述为“正式逐镜生成”“正式视频生成”或“进入批量生成”；只能建议补齐资产、整理已有测试帧、生成/抽取关键帧，或进入资产确认门。

允许进入正式视频生成的最低条件：

1. `10A-已确认资产基准表` 中已有本阶段必需的产品、主要角色、核心场景/首帧和文字道具。
2. 缺口已在项目资产清单中明确降级为“非本阶段必需”“备用生成”或“可无首帧先测”。
3. 用户已经明确确认当前资产基准可用于正式生成。

如果用户问“下一步做什么”，且高优先资产未齐，默认回答必须是“先补齐剩余高优先资产”，并列出缺口；不得提前建议正式逐镜生成登记。

## 分镜脚本修改同步门槛

每次创建、插入、删除、合并、拆分、顺延或改写分镜视频脚本时，必须同时满足三项要求：

1. 增加或保留清晰时间信息：年代/成长阶段、日夜或具体时段、季节/天气、角色年龄层或身份状态。若时间无法确定，必须用安全假设标注。时间信息必须写进可复制的视频提示词代码块，包含建议生成总秒数和分秒动作轴，不得只写在小节说明里。
2. 增加或保留符合剧情因果的逻辑引导：本镜承接上一镜的原因、角色为什么做这个动作、动作完成后为什么触发对白或反应，以及它如何推动下一镜。
3. 同步修改项目中受影响的对应内容：润色视频脚本、逐镜操作单、合并版操作单、镜头合并拆分建议、资产清单、生成记录、状态页、镜头编号、备用插入、回退引用、叠化/回扣引用和任务总数。若某个关联文件暂不修改，必须明确说明原因和剩余风险。

修改完成后必须做项目级对应关系检查：镜头编号不得重复，新增镜头必须有原镜头编号或新增编号，后续编号和引用必须同步顺延，项目状态中的任务总数必须与操作单一致。

## Seedance 网页端提示词执行结构

`ABD-PROD-2026-002-houlai-mama` 已完成 L2 原创生产闭环验证。后续项目进入 `12-即梦网页端逐镜操作单` 时，默认采用该项目沉淀出的执行结构，而不是散文化导演说明。

可复制提示词必须按以下优先级组织：

```text
生成规格/时长/画幅 → 本镜核心目标 → 场景时间 → 参考资产 → 角色锁定 → 单一镜头控制 → 声音白名单或无对白规则 → 逐秒时间线 → 剪辑手柄和尾帧 → 禁止项
```

执行要求：

1. “本镜核心目标”必须说明本镜在全片中负责钩子、压制、反转、揭示、承接、回扣还是留白。
2. 逐秒时间线必须写进代码块，推荐使用 `0-2秒...；2-3.5秒...；3.5-...秒...` 的连续分号结构。
3. 对白镜头必须写清说话对象、视线、身体朝向、开口触发条件、台词白名单和非说话者嘴部状态。
4. 有道具、手机、便签、产品或按钮动作时，必须写清起点、移动方向、归属、完成态和尾帧状态。
5. 网页端提示词可以继承 002 的执行方法，但不得继承 002 的具体题材、人物关系、台词、场景组合或产品信物逻辑。
6. 导演母版可以保留更完整的剧情解释；网页端代码块必须转译为模型执行语言。

## 后续扩展流程

```text
可交付 → 推荐高风险镜头测试（用户决定是否执行）→ 正式逐镜生成 → 粗剪 → 成片审核
→ 发布合规 → 已发布 → 数据复盘 → 训练升级
```

这些状态保留为系统扩展接口。当前可以人工记录和复盘，但不作为系统已经自动操作网页、自动剪辑或自动发布的能力。高风险测试是系统推荐的降风险步骤，不是用户手动生成全片素材的强制前置；如果用户决定跳过测试，应按逐镜任务包直接生成，并在失败时按单变量重试记录复盘。

## 最小训练闭环

1. 按新手模式分析参考视频，只生成候选结论。
2. 系统主动给出`建议学习`、`谨慎学习`、`不建议学习`和`禁止照搬`四类判断。
3. 用户确认、删减或修改推荐项；用户不需要先掌握视频专业术语。
4. 通过归档 Skill 将确认内容写入对应训练库，初始为L1。
5. 结合真实产品或新故事完成原创迁移，用户满意后升级为L2。
6. 记录生成失败和单变量重试，用结果修正提示词规则与避坑经验。
7. 发布后使用真实数据验证，成立的能力升级为L3，不成立的规则降级或标记待复核。

自动优化仅包括分析、比较、推荐、候选总结和基于证据的规则修正。未经用户明确确认，不得把候选结论提升为正式正向训练知识。

## Seedance 2.0当前生产闭环

1. 使用简单文案输入模板记录原稿和允许调整范围。
2. 逻辑初审输出原稿保留版、微调版和按需提供的生成友好版。
3. 用户选择A/B/C/D/E，确认后产生唯一锁定文案。
4. 使用锁定文案前先确认发布平台、画幅比例、目标片长和当前 Seedance 入口参数；未确认时只能生成“画幅待确认”的导演分镜，不得写死生产画幅。
5. 使用锁定文案编译导演分镜、资产缺口、关键帧计划和逐镜任务包。
6. 用户确认角色、场景、产品、资产和分镜。
7. 先输出核心资产生产清单，并优先生成或整理核心资产：主角、产品、核心场景和关键首帧。
8. 只有核心资产硬门槛满足后，才输出或更新每镜独立可复制的 Seedance 网页端操作单，包括上传资产、首帧策略、模型、画幅、时长、分辨率、运动幅度、对白/音频、完整提示词、禁止项、验收标准和失败降级方案。
9. 系统推荐身份、最复杂动作、产品/文字等高风险镜头测试；用户决定是否测试。若用户跳过测试，也必须先满足核心资产硬门槛，才可按 `SHOT_00P/SHOT_01...` 顺序或系统推荐顺序正式逐镜生成。
10. 如用户提供生成结果，系统先抽帧看画面；若有可见口型或音轨风险，按 `音画同步检测流程_v01.md` 做基础或 speech ROI 检测；再记录失败类型、单变量修改和下一镜参数。
11. 已经在同项目实测通过的低运动关系动作、静态手部留白、无精确产品/文字镜头，可按“无首帧测试分级规则”先无首帧生成或测试一版；产品标准露出、精确文字、严格站位和复杂动作仍优先生产型首帧。该规则不得绕过核心资产硬门槛。

## 阶段交接提示规则

每次执行到人工确认门或阶段交接点时，必须显式提醒用户下一步应该如何操作，包括：

1. 当前阶段是否已经完成。
2. 用户需要确认、修改、测试还是暂停。
3. 如果确认通过，下一步进入哪个具体阶段。
4. 如果建议测试，说明优先测试哪些高风险镜头或资产，并明确用户可以选择跳过测试直接正式生成。
5. 不得只交付文件或方案而不说明下一步。

推荐句式：

```text
当前已到 Gx 确认门。建议你先确认 A/B/C；确认后下一步是 D；如果暂不确认，则应先修改 E。
```
