# 脚本与分镜制作

## 阶段一：脚本

1. 确认片长、画幅、平台和核心情绪。画幅、平台和片长必须由用户明确确认；未确认时只能写“待确认”或列出候选，不得默认 9:16、16:9、抖音、横屏或竖屏。
2. 确认脚本来源：原创方向、参考结构迁移、活动命题、产品故事或用户已有文案。若来源包含参考视频，先保留学习到的结构机制和情绪曲线，避开参考片人物、台词和完整情节链。
3. 若脚本来自系统创意，必须已有G0爆款方向门和8秒开场测试结果；若缺失，先退回 `create-video-concept` 或补做G0，不直接写完整分镜。
4. 写出前3秒钩子、前8秒停留承诺、人物目标、冲突升级、产品进入、高潮和结尾互动。
5. 建立情绪阶梯，至少包含轻冲突、现实处境、关系代价、不可逆遗憾或主题回扣中的3层；控制每段信息增量，删除只抒情不推进的内容。每个情绪段都必须有可见物件、动作或空间变化承载，不得只靠旁白解释。
6. 逐句检查关键台词：短、准、藏、重、可演。角色说话必须精简但有引人深思的余味；哲理藏在说话对象、未说出口的代价、物件和停顿里，不由角色直接喊出口号。
7. 对转折段、高潮段和结尾回扣做“功能检查”：这段负责铺垫、递进、反转、释义还是邀请互动；如果功能不清，先改功能再润色语言。
8. 使用评分表检查脚本，并等待用户锁定。未经用户锁定，不进入逐镜分镜或 Seedance 编译。

### 爆款叙事硬门槛

命题短片、情绪短片、参考爆款迁移、群像答案型文案、亲情、友情、遗憾、成长或错过主题，在分镜前必须已通过“停留-递进-故事-呼应”验收：

1. 前 3-8 秒有停留钩子。
2. 前 20 秒有继续观看理由。
3. 中段至少 3 次情绪升级。
4. 每段有可见故事单元，不是并列金句。
5. 结尾回扣开头同一动作、物件、台词、问题或关系。

若未通过，先退回脚本文案重构，不制作分镜。

额外台词要求：

1. 每句关键台词必须有明确说话对象。
2. 不用长台词解释人物情绪，优先用动作、停顿和物件承接。
3. 最重的一句话通常越短越好，必须让观众能从上下文读出更深的人生哲理。
4. 若角色台词直接讲道理，应退回脚本阶段改成处境化表达。

### 答案型文案分镜前检查

街采、群像回答、多人依次回答同一问题、答案池或金句型结构，在分镜前必须已完成说话关系表：

```text
角色身份 → 说话对象 → 缺席关系 → 未说出口的代价 → 画面锚点 → 人生哲理
```

如果缺少该表，或角色只是在对观众讲道理，应退回脚本阶段，不制作分镜。

## 阶段二：分镜

脚本锁定后，输出逐镜表：时间、景别、构图、人物动作、台词、旁白、声音、镜头运动、转场、产品要求、生成难度和本镜的剧情功能。

分镜前必须确认发布平台、画幅比例和目标片长。若用户尚未确认，分镜文件必须标为“画幅待确认版”，每个涉及构图的说明只能描述相对构图和候选画幅影响，不得写死生产画幅。

分镜前必须再检查：

1. 钩子是否用具体问题、动作或冲突打开，而不是抽象价值观。
2. 情绪是否层层递进，每段答案或事件都比上一段多一个新维度。
3. 高潮是否来自前文铺垫的物件、动作或选择，而不是突然出现的金句。
4. 结尾是否能首尾呼应，并把问题交还给观众。
5. 任何需要 Seedance 生成人物说话的镜头，是否已经保留台词、说话对象、非说话者状态和无字幕/无BGM要求。
6. 任何关键台词是否仍然精简、有对象、有未说出口的代价，并能由画面承接；不合格台词先改文案，不直接进入视频提示词。

## 制作约束

- 减少难以保持一致的角色、服装和场景数量。
- 对产品外观、手部动作、多人互动和口型标记高风险。
- 先设计角色正脸、复杂动作和产品特写三个测试镜头。

## 交接

脚本和导演分镜锁定后，交给 `compile-seedance2-shots` 生成资产计划、关键帧计划、多模态引用分配、逐镜提示词、验收和重试方案。本Skill不重复维护Seedance平台参数。

如果用户已有基本完整的文案，只希望检查逻辑、反转、对白长度或产品植入，不需要大幅重写，优先转交 `refine-video-script-logic`。
