# 参考视频分析

## 工作流程

1. 获取视频，确认时长、画幅、来源和可见字幕。
2. 按内容密度每2至5秒抽帧，基于真实画面分析。
3. 重建钩子、人物目标、冲突、信息差、情绪递进、高潮和回扣。
4. 分析景别、机位、运镜、转场、构图、光影、色彩、声音、表演和剪辑节奏。
5. 标记产品首次出现时间、功能展示和品牌露出。
6. 对照当前能力地图，识别新增能力、重复能力、冲突规则和训练缺口。
7. 区分题材与结构，禁止只给原故事替换产品。
8. 对每个可学习点做联想扩展：至少列出 3 个可替代题材、关系或行业场景，说明它学习的是机制而不是原片母题。
9. 输出传播原因、制作难点、Seedance生成难度、版权肖像风险和原创边界。
10. 对照已有训练库，完成相似度、增量和冲突判断，推荐 A/B/C/D 或合并方案。
11. 将结论标记为候选，用户确认后再交给归档 Skill。

## 联想扩展与默认母题隔离

参考视频分析的目标是提炼可迁移机制，不是把原片题材、人物关系或情绪包装变成默认母题。每次分析必须主动完成“从案例到机制”的拆解：

1. `可学习机制`：写清真正值得学的是钩子、信息差、重复动作、角色选择、产品进入、视听节奏、情绪反转还是传播话题。
2. `可替代场景`：至少联想 3 个不复制原片人物关系的迁移方向，例如亲情样本也要联想到友情、职场、社区、陌生人互助、职业传承、老板IP观点或公共议题。
3. `不可默认母题`：列出不能被系统自动套用的表层元素，例如妈妈、母女、失忆、离世、遗物、贫困、老人辛苦、儿童受伤、特定职业、特定产品道具或原片台词。
4. `近期冷却提醒`：如果该案例与最近已做项目在人设、疾病/遗忘机制、离别方式、物件信物或结尾回扣上相似，必须标记为“近期不建议作为默认方向”，只能作为候选变体或反例边界。
5. `原创迁移优先级`：优先推荐机制级迁移，其次是场景级迁移；人物关系和完整情节链默认不迁移，除非用户明确要求。

输出学习建议时，不得只写“可以学习亲情/可以学习妈妈照顾/可以学习遗忘”。必须写成“学习 X 机制，可迁移到 A/B/C 场景；原片的 Y 元素不得默认复用”。

## 视频文案提取模式

当用户要求“拉取视频文案”“提取文案”“整理视频定位、主题、场景、角色设定和对话”时，优先执行轻量文案提取模式；除非用户同时要求训练判断，否则不强制输出完整 A/B/C/D 入库矩阵。

1. 先确认视频文件、时长、画幅、抽帧间隔和证据范围。
2. 基于真实画面和可见硬字幕整理文案；如果没有可靠音频转写，不得假装获得完整人物台词。
3. 人物统一匿名为 `A/B/C...`，避免复用原片可识别人物设定。
4. 输出字段至少包括：
   - `视频定位`
   - `核心主题`
   - `场景结构`
   - `角色设定（A/B/C...）`
   - `对话（A/B/C...）`
   - `一句话提炼`
   - `证据边界 / 使用边界`
5. `角色设定` 不使用表格，逐行写成冒号格式：
   - `A：...`
   - `B：...`
   - `C：...`
6. 只要视频中出现人物声音或人物说话，必须为该说话者分配 `A/B/C...` 角色编号，并在 `角色设定` 中补出身份、关系和叙事功能。不得用简单的 `对白：...`、`旁白：...`、`字幕：...` 或无归属文本回避人物归属。
7. `对话` 不使用表格，逐行写成冒号格式；人物说话统一使用 `A：...`、`B：...`、`C：...`。本章节只写人物发声或人物讲话，不使用 `旁白：...`、`字幕：...`、`对白：...` 标签。
8. 纯画面字卡、屏幕文字、标题、标语、证件/奖牌可见文字不写入 `对话` 章节；如确有保留价值，另设 `画面文字 / 字卡证据` 章节记录，并明确它不是人物台词。
9. 声音存在但暂时无法从画面、ASR 或上下文判断说话者时，不得直接写入最终 `对话`；先回看该时间段画面，仍无法判断时放入 `待复核片段`，写明时间段、候选文本和需要复核的问题。
10. 对新出现的人物声音，先根据视频文案和画面推导其身份，再编号。例如：婆婆、单位领导、监狱工作人员、医生/护士、摊主母亲、顾客、路人等都应成为独立角色或合并为功能性群像角色。
11. 角色编号要稳定：同一人物或同一功能性群像在全文只使用一个编号；不要同一人前后换编号。
12. 对话要区分 `确认可见字幕`、`画面推断` 和 `待音频核对`。不得把推断内容写成确定台词。
13. 若用户要求交付文件，默认在该视频对应项目目录内输出 Markdown；只有用户明确要求 Word/PDF 时才生成对应格式。
14. 该模式的产物是参考文案或训练候选，不等于正式正向知识入库；入库仍需用户明确确认。

### 精准提取与节省 token

视频文案提取必须按“少量概览 + 定点复核”执行，不要把所有抽帧内容一次性塞进上下文。

1. 如果用户允许本地算力，先跑本地 ASR，得到带时间码的对白候选；ASR 只作为补漏、切分和说话人判断辅助，不直接当事实。
2. 首轮用 2-5 秒抽帧建立人物、场景和主结构，只读接触表，不逐张展开所有帧。
3. 正式提取字幕时，优先按“字幕变化”而不是 ASR 分段取样：
   - 先人工或程序确认常驻对白字幕基线，例如底部字幕的纵向位置。
   - 在字幕基线上按 0.2-0.3 秒高频裁剪字幕窄带，覆盖所有可能出现的短句。
   - 对相邻裁剪图做图像去重或相似度聚类，只保留每条字幕最清晰的一张；不要把重复帧当成重复台词。
   - 若本地 OCR 可用，先对去重后的字幕窄带做 OCR，生成 `时间段 -> 字幕候选` 表；OCR 结果仍需和画面/ASR校对。
   - 若本地 OCR 不可用，也要先生成去重接触表，再人工/模型读取接触表；不要回退到只按 ASR 时间点抽帧。
   - 如果去重后候选仍超过约 80 条，说明聚类或 OCR 不足；不得把全部候选图直接塞入上下文。应继续收紧聚类、安装/启用本地 OCR，或按疑似漏句时间段分批复核。
4. 发现字幕密集、快切、模糊、小字、疑似错字或漏句时，对对应时间段提高到 0.1-0.2 秒局部复核。
5. 复核时优先制作“字幕精准裁剪接触表”，按每条字幕的实际字形位置动态裁剪，不再默认截取“字幕上下三行”或大面积画面。裁剪要求：
   - 先定位硬字幕所在区域；常见底部字幕只保留字幕文字高度和少量上下安全边。
   - 安全边建议为字幕字高的 15%-30%，或上下各 8-20 像素；不要把人物主体、背景大面积带入裁切图。
   - 如果字幕有描边、阴影、发光或运动模糊，安全边可以略放宽，但仍以完整识别文字为目标。
   - 同一时间点可保留时间码或文件名标记，但时间码不得遮挡字幕。
   - 遇到中上部字卡、证件、奖牌、弹幕式文字时，按该文字所在位置单独精准裁剪，不套用底部字幕区域。
   - 如果自动亮度/白字检测误抓灯光、衣服、反光物或背景高光，立即回退为“固定字幕基线窄带裁剪”：先人工确认字幕常驻纵向位置，只截该窄带并按需放大，不继续扩大整张画面。
6. 非底部文字必须单独扫描：
   - 中部/上部字卡、黑底金句、证件、奖状、文件、店招、手机屏幕等不套用底部字幕基线。
   - 每 0.5-1 秒低频扫描全画面或关键区域，发现文字后按文字所在区域单独精准裁剪。
   - 这些内容默认进入 `画面文字 / 字卡证据`，只有确认是人物发声才进入 `对话`。
7. 用 OCR/硬字幕候选、ASR 文本和最终文案做三表差异校对：
   - ASR 有、字幕抽帧稿没有：作为漏句候选补入。
   - 字幕有、ASR 听错：以清晰画面字幕为准。
   - OCR/硬字幕候选有、最终文案没有：必须说明是重复、误识别、画面文字，还是漏补。
   - 最终文案有、OCR/硬字幕和 ASR 都没有：必须标明来源，不能写成确定台词。
   - ASR 补出的说话内容必须回填到具体人物编号；无法定位时先打开该时间段画面确认说话者，仍无法确认时放入 `待复核片段`，不要混入最终 `对话`。
   - 金额、刑期、年份、证件、奖牌、职务、地名：必须用画面关键帧或多证据确认。
   - 屏幕字卡、证件、奖牌和标语：不得被 ASR 覆盖。
8. 只对裁切图里仍看不清的字卡、证件、奖牌、金额、刑期、年份等高风险内容打开原始关键帧。
9. 只记录新增/修正句子，不重复描述已确认画面。
10. 对低置信句子不要放入最终 `对话`，统一放入 `待复核片段` 或 `修正清单`，并写清时间段、疑点和下一步核对方式。
11. 输出文案前做一次“错字/漏句检查”：
   - 人名匿名是否一致。
   - 数字、金额、年份、刑期、职业、地名是否有画面证据。
   - 口语句是否因字幕遮挡被误读。
   - 字卡、标语、证件、奖牌等屏幕文字是否被误当成对白。
   - OCR/字幕候选、ASR 候选、最终对话三者是否都已对齐；未采用的候选是否有原因。
12. 修改旧文案时，优先生成递增版本文件，并只输出修正清单和更新后的文件；不要重讲完整分析，除非用户要求。

## 历史库比较

只读取完成本次判断所需的索引、卡片和登记表，不加载全部完整案例分析。

1. 先读 `00-系统总控/训练样本登记表.md`、`当前能力地图.md` 和相关库文件名。
2. 根据新视频候选标签，定向读取最相近的案例卡、结构卡、钩子卡、偏好卡和避坑卡。
3. 分别比较：
   - `题材相似度`：人物、行业、时代、产品和议题是否相近。
   - `结构相似度`：钩子、冲突链、案例递进、高潮、反转和闭环是否相同。
   - `视听相似度`：镜头、色彩、表演、声音和剪辑节奏是否相近。
   - `新增能力`：是否出现已有库中没有、且可独立迁移的方法。
   - `冲突规则`：是否与已确认偏好或避坑规则相反，以及两者能否按适用场景并存。
4. 不使用虚构的精确百分比。使用 `高度相似 / 部分相似 / 明显不同`，并列出画面或结构证据。

## A/B/C/D 推荐矩阵

### 推荐 A：完整入库

同时满足：

- 至少两个维度具有明确长期学习价值，例如结构加情绪、钩子加植入或视听加表演。
- 相比现有库存在实质新增能力，或能用不同题材交叉验证关键方法并补充适用边界。
- 用户可能将其作为稳定审美或导演路线，而不只是喜欢一个具体故事。
- 原创边界、适用场景和生成风险可以说清楚。

已有高度相似结构时，不得仅因题材或人物不同推荐 A；必须说明新增了什么。

### 推荐 B：只存案例

符合任一情况：

- 视频值得保留，但主要是在重复验证已有结构，没有独立的新结构或新偏好。
- 效果依赖具体演员、热点、名人、真实事件或特殊题材，可迁移性有限。
- 证据充分但规律尚未被多案例验证，适合作为补充样本。
- 新视频可增加正例、反例或适用边界，但不值得新增结构卡。

### 推荐 C：保存为新结构模型

同时满足：

- 最值得学习的是叙事骨架，而不是原题材、表演或整体审美。
- 与现有结构存在机制级差异，不只是改名、换题材或调整顺序。
- 能明确写出步骤、每步作用、适用场景、禁用场景和原创迁移方式。
- 结构可以脱离原人物与事件，在至少两个不同题材中合理成立。

若只是已有结构的变体，推荐合并扩展已有结构，不新建结构模型。

### 推荐 D：暂不入库

符合任一情况：

- 无法获得足够画面、字幕、声音或上下文证据。
- 用户的喜欢点尚不清楚，正负判断冲突且无法按场景拆分。
- 传播效果主要来自争议、误导、侵权或不可迁移的偶然因素。
- 与当前训练目标无关，或需要先经过原创/生成测试才能判断。
- 新视频没有为已有知识增加案例价值、结构价值或边界价值。

## 合并决策

在 A/B/C/D 之外主动检查是否应合并：

- `合并案例证据`：结构高度相似，只增加一个验证案例。推荐 B，并回链已有结构卡。
- `扩展已有结构`：核心机制相同，但新增步骤、适用场景、失败条件或情绪层级。推荐合并更新结构卡，不新建同义卡。
- `合并钩子`：只是同一钩子的不同台词或题材表达，加入变体示例。
- `扩展审美偏好`：偏好相同但增加适用场景；不要重复创建偏好卡。
- `记录条件化冲突`：两种风格都有效但用途不同，为各自标注适用条件，不互相覆盖。
- `保留独立结构`：故事步骤、观众信息差或价值翻转机制本质不同，才建议新建。

合并只是一种推荐。未经用户确认，不得修改已有正式卡片。

## 默认新手模式

即使用户只上传视频、没有说明喜欢点，也必须主动判断并解释：

- `建议学习`：效果有证据、可复用、适合当前目标且具有原创迁移价值。
- `谨慎学习`：有效但依赖高成本、真人表演、复杂生成、特定平台或特定题材。
- `不建议学习`：低效、过时、与品牌冲突、难以验证，或只是表面形式而没有叙事作用。
- `禁止照搬`：完整故事链、具体人物、独特台词、品牌资产、音乐、受保护画面或高度识别性的创意表达。

每个判断都要使用通俗语言说明证据、适用场景、风险和原创迁移方式。不要要求新手先使用专业术语，也不要因为用户没有偏好就省略推荐。

## 自动优化边界

- 可以自动完成分析、横向比较、候选训练点排序、重复知识合并建议和验证方案。
- 可以根据原创迁移、生成失败和发布数据提出规则升级、降级或待复核建议。
- 不得自动批准正向知识入库；未经用户明确确认，候选结论不得成为稳定规则。
- 失败经验与未验证想法必须和正向案例分开记录。

## 必须输出

- 客观信息和证据范围。
- 逐段结构与情绪曲线。
- 镜头、转场、光影、声音、表演和剪辑判断。
- 产品植入逻辑。
- 四类学习建议及其理由。
- 可复用能力、原创迁移方式与不可照搬内容。
- 联想扩展：每个主要学习点的可替代题材/关系/行业场景，以及不可默认化的原片母题。
- 与已有训练能力的差异。
- 历史库相似项、实质新增项、冲突项和建议合并目标。
- A/B/C/D 或合并推荐、置信度、理由和不选其他项的原因。
- 推荐的下一步：放弃 / 入库确认 / 原创迁移测试 / 生成可行性测试。

如果无法获得视频画面，必须明确证据不足，不得假装完成逐帧分析。

## 输出收尾强制规则

每次分析参考视频时，无论用户是否主动要求“推荐”，最终回复都必须以“分析建议 / 推荐选择”收尾，并给出：

1. 推荐选项：A 完整入库 / B 只存案例 / C 新结构模型 / D 暂不入库 / 合并到已有知识。
2. 置信度：高 / 中 / 低。
3. 推荐理由：用 2-4 条说明为什么选择该项。
4. 不选其他项的原因：至少说明为什么不选最容易混淆的 1-2 项。
5. 用户确认提示：提醒用户只有明确确认后才写入正式训练库。

如果本次只是“提取文案”，仍需提供轻量推荐；可简写，但不得省略。
