# AiBianDao 0-1 实施路线图

## 阶段零：理解系统

先记住：

- 项目文件夹保存长期资料。
- Skill 保存可重复的工作方法。
- 对话框执行一次具体任务。
- 训练库保存用户确认后的能力。
- 总控只管理方向和状态，不承担所有重任务。

## 第一天：完成基础资料

1. 填写品牌定位。
2. 填写目标用户。
3. 复制产品卡模板，建立第一个产品卡。
4. 在项目看板登记第一个正式生产任务，编号使用 `ABD-PROD-YYYY-NNN-short-name`。
5. 新建“参考片01”对话框。

完成标准：AI能够准确回答品牌是谁、产品卖给谁、哪些内容不能出现。

## 第一周：训练分析和原创迁移

选择3至5条风格不同的优秀参考视频，每条单独对话：

1. 先只分析，不直接入库。
2. 用户补充喜欢和不喜欢。
3. 每条提炼一个真正不同的新能力。
4. 至少选择一个结构完成产品原创迁移。
5. 用户满意后再正式归档。

完成标准：训练库拥有至少3个L1案例、1个L2结构和明确的第一批审美偏好。

## 当前第一优先级：完成首条 Seedance 2.0 正式任务包

1. 建立 `ABD-年份-编号-名称` 项目目录。
2. 填写需求与状态。
3. 用户已有具体简单文案并要求审稿或生产时，不强制生成三个原创方向；用户要求系统从主题、产品、命题或参考片创作文案时，仍必须先给 3-5 个候选方向。
4. 完成逻辑初审并由用户锁定唯一文案。
5. 完成导演脚本、分镜、角色、场景和产品外观确认。
6. 生成资产清单、关键帧计划和每镜独立的 Seedance 逐镜任务包。
7. 运行任务包校验，并交付手动生产顺序。
8. 用户手动生成后，如提供结果，再记录高风险测试和失败重试。
9. 剪辑、发布和数据复盘作为后续人工扩展步骤。

当前完成标准：至少一个项目从简单文案走到通过校验的逐镜任务包，没有多个冲突的生效版本。后续再把完成标准扩展到成片与数据复盘。

## 前五个项目的目标

前五个项目主要用于验证流程，不追求完全自动化：

- 项目1：验证参考分析到原创脚本。
- 项目2：验证虚拟角色资产和跨镜一致性。
- 项目3：验证产品植入、包装准确性和低运动特写。
- 项目4：验证不同平台剪辑和包装。
- 项目5：使用数据复盘修改已有结构。

重复出现两次以上的问题，才考虑新建规则或拆分新 Skill。

## 重复工作识别与Skill治理

系统在执行日常任务时主动识别重复步骤，并在以下情况提醒用户评估沉淀：

1. 同类工作已经出现至少两次。
2. 输入、输出、步骤和验收标准逐渐稳定。
3. 每次都需要重复解释同一套非通用知识或操作方法。
4. 该流程容易遗漏、出错，或需要固定工具和脚本保证一致性。

提醒时按以下优先级给出建议，并由用户决定是否实施：

1. 写入项目总规则：适用于全局边界、审批门和默认行为。
2. 更新模板或知识卡：适用于固定字段、检查清单和可复用知识。
3. 扩展已有Skill：适用于与现有Skill属于同一任务阶段的新增步骤。
4. 新建独立Skill：仅适用于边界清楚、可单独触发、输入输出稳定且不会与现有Skill重叠的完整工作流。

“识别重复工作并建议如何沉淀”本身属于系统治理规则，不单独建立Skill。系统可以自动发现并提醒，但新建、拆分或合并Skill前必须获得用户确认。

## 什么时候扩展系统

满足以下条件后，再增加自动资产生成、自动剪辑、平台适配和数据复盘 Skill：

1. 已完成3至5个真实项目。
2. 当前步骤经常重复且方法稳定。
3. 能明确写出输入、输出和验收标准。
4. 新 Skill 不会与已有 Skill 职责重叠。
5. 不能仅靠一条总规则、一个模板或扩展现有Skill解决。

## 每次结束任务前检查

- 最终成果保存在哪里？
- 当前项目状态是否更新？
- 用户是否明确确认？
- 是否有值得归档的新能力？
- 是否记录失败原因？
- 下一步是谁做什么？

全部回答清楚后，再归档对话框。

## 新手阶段不要做

- 不要一次安装或创建十几个空 Skill。
- 不要把所有案例全文塞进总控对话框。
- 不要未经确认就自动训练入库。
- 不要一开始追求全自动生成和自动发布。
- 不要只看点赞，必须结合留存、完播、转发和转化。
- 不要把参考片的故事换产品后称为原创。
