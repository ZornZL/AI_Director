# 工作流程

## 路由

### 从主题开始

```text
主题/大赛规则/产品命题
→ 提取硬规则和禁止项
→ 3个差异明显的创意方向
→ 用户锁定方向
→ 完整脚本
→ 逻辑审核与锁定
→ Seedance生产编译
```

### 从具体脚本开始

```text
用户原稿
→ 原稿保留
→ 逻辑、合规和生成风险审核
→ 用户选择A/B/C/D/E
→ 唯一锁定文案
→ Seedance生产编译
```

## 新手六步

1. **主题输入**：确认规则、目标、产品、平台和大致时长；未知项标为待确认。
2. **创意选择**：只给3个方向和开场，不在用户选择前写完整正片。
3. **脚本锁定**：生成完整脚本，用户用A/B/C/D/E确认。
4. **导演分镜**：确认平台、画幅、总片长后，拆成独立4—15秒任务。
5. **资产与任务**：确认核心人物、场景、产品、文字道具和必要关键帧；输出每镜可复制任务。
6. **测试与复盘**：先测身份、最难动作、产品/文字；失败后一次只改一个变量。

## 可选故事版

超过6镜、跨场景、跨年龄、人物一致性或复杂动作重要时，可以在正式资产前增加故事版预审。故事版仅用于检查，不是强制门，也不替代正式资产、首帧或Seedance任务。

## 每镜最低交付标准

- `SHOT_ID`、剧情功能、建议4—15秒时长和画幅。
- 明确时间、场景、角色年龄与前镜承接。
- 引用哪些人物、场景、产品、文字或关键帧资产。
- 一个主要机位和一个主要运镜意图。
- 起始状态、逐秒动作、动作终态和稳定尾帧。
- 谁说话、何时说、只允许哪些声音；非说话者状态。
- 连续性、禁止项、验收标准和失败降级。

## 已验证规则

来自002闭环：先停稳和对视再说话；明确视线对象；手机先决定给谁看；道具写清起点、路径、接收者和终态；锁定递接所有权；多年龄角色分资产；对白使用白名单；复杂镜头失败时依次简化、缩短、拆镜或补关键帧。

## 人工确认门

- G0：创意方向。
- G1：锁定脚本。
- G2：平台、画幅、总片长和导演分镜。
- G3：核心外观、资产与关键帧。
- G4：高风险测试和逐镜任务包。
- G5：粗剪与发布前合规。

每次到达确认门，必须告诉用户：需要确认什么、下一步是什么、确认后会生成什么。
