# Seedance 2.0 逐镜编译器

把用户已经确认的锁定文案编译成完整视频脚本和可执行的逐镜任务。当前这是系统唯一稳定承诺的生产能力，不把未经确认的原始文案或候选修改直接当作视频提示词。

## 边界

- 所有视频镜头均由 Seedance 2.0 生成，不输出现实拍摄方案。
- 摄影、机位、运镜、灯光、表演、走位、场景和道具描述，全部用于控制 AI 生成结果，不得写成摄影师、演员或现场剧组执行指令。
- 不把现实演员、摄影设备、实体场地、现场布光和实拍预算列为生产前置条件。
- 只面向 Seedance 2.0；能力参数以 `references/capability-matrix.md` 为准。
- 首帧策略以 `references/asset-keyframe-rules.md` 的“首帧风险分级”和“无首帧测试分级规则”为准：正式生产仍以首帧锁定关键状态，但同项目实测已验证的低运动关系镜头、静态手部镜头和无精确产品/文字镜头，可以先无首帧生成或测试一版。
- 保留用户确定的人物、关系、台词、事件顺序和结局。
- 所有补充内容分为“安全假设”“生产假设”“阻塞缺失”。
- 未经确认不得偷偷改写台词或剧情；优化建议放在“未采用建议”。
- 文案存在未确认的逻辑调整时，先调用 `refine-video-script-logic`；只接收用户明确锁定的版本。
- 不承诺“完美一次生成”，目标是可执行、可验收、可降级重试。
- 真人、商标、音乐和参考作品先通过合规检查。
- 默认用户在即梦或当前 Seedance 网页入口手动生成视频；Codex 负责先提供核心资产和每镜所需资产、首尾帧、网页端参数、可复制提示词、验收标准和复盘建议，不假装已经生成视频。
- CLI 只作为图片资产、关键帧、查状态或辅助下载的候选工具；在当前系统中暂不把 CLI 作为默认视频生成入口。若用户明确要求用 CLI 生成视频，必须先说明可能排队很慢、会消耗额度，并取得确认。
- 编译阶段不要求用户先经过创意 Skill；但这只适用于用户已经提供可识别的简单文案或剧情原稿，并且目标是审稿、分镜或生产。若用户要求系统从主题、产品、命题、参考结构或空白需求开始创作文案，必须先转交 `create-video-concept` 输出多方向候选并等待锁定，不得直接生成唯一完整文案。
- 核心资产硬门槛：主角、产品、核心场景、关键文字道具和高风险镜头首帧中的高优先资产未生成、未整理或未在 `10A-已确认资产基准表` 中确认前，不得建议进入正式逐镜生成、批量生成、正式视频生成登记或正式网页端生成。只能建议补齐资产、整理测试抽帧、进入资产确认门或明确降级缺口。
- 分镜脚本修改同步门槛：每次创建、插入、删除、合并、拆分、改写或顺延分镜视频脚本时，必须同时补齐“时间/年代/时段”和符合剧情因果的逻辑引导。这里的时间必须写进可复制的视频提示词代码块，包含建议生成总秒数和分秒动作轴，不得只写在小节说明里。不得只改单段提示词；必须检索并同步项目中所有受影响的对应内容，包括润色视频脚本、逐镜/合并操作单、镜头合并建议、资产清单、生成记录、状态页、镜头编号引用、回退引用和后期回扣引用。若某个关联文件暂不修改，必须说明原因和剩余风险。
- 单镜头提示词骨架：每个可复制提示词必须显式包含角色锁定、场景时间、动作闭环、单一摄影意图、音画同步、可见风格、情绪节奏、逐秒时间轴和剪辑手柄。该骨架只用于强化单镜头执行，不替代资产、关键帧、验收、连续性和失败降级。
- 已验证生产项目方法论迁移：`ABD-PROD-2026-002-houlai-mama` 的 Seedance 生产闭环已验证，正式网页端可复制提示词必须优先采用“硬约束前置 + 本镜核心目标 + 资产/角色锁定 + 单一镜头控制 + 声音白名单 + 逐秒时间线 + 稳定尾帧 + 禁止项收束”的模型执行结构。面向人的导演母版可以保留更完整的剧情解释，但进入 `12-即梦网页端逐镜操作单` 的提示词不得写成散文化导演说明，也不得只堆抽象风格词。
- 长期默认声音与字幕生产口径：所有新项目默认采用“无字幕、无BGM分镜生成”。Seedance 分镜视频默认不生成字幕、片名、黑屏字卡、结尾文案、水印，也不生成背景音乐、配乐、歌词、背景声铺底或环境音乐。Seedance 阶段必须保留人物说话声音，以及人物场景中真实发生且应当发出的剧情内声音，例如脚步、门响、敲门、手机语音、碗筷声、物件碰撞、雨打伞面等。字幕、片尾字卡、背景音乐、环境氛围铺底和整体声音包装默认在后期剪辑/发布阶段添加。若某项目要求全部对白也后期配音，必须在项目声音圣经中明确。
- 发布平台与画幅确认门：正式逐镜任务包前必须已有用户确认的发布平台、画幅比例、目标片长和当前 Seedance 入口参数。未确认时不得输出可执行网页端参数，不得默认 9:16、16:9、抖音、横屏或竖屏；只能输出候选参数和需要用户确认的问题。
- 故事版是未验证的可选预审。只有用户明确要求，或复杂项目需要先看连续画面时才提供；不得把它写成强制门，也不得替代正式资产、首帧或逐镜任务。

## 工作流程

### 0. 生成后闭环复盘

当用户提供已经生成的视频、实际粘贴到 Seedance 的提示词、重试提示词或人工反馈时，先进入生成后复盘模式，不要直接重新编写全片。必须区分：

- 原始操作单提示词：用于判断编译器初始输出。
- 实际粘贴提示词：用于判断 Seedance 真实执行输入。
- 重试提示词：用于判断哪些变量被修改以及是否有效。
- 生成视频或抽帧：用于判断模型真实画面，不用文本想象替代。
- 用户满意/不满意反馈：用于确定创作者验收口径。

复盘输出必须包含：逐镜提示词-视频对应表、成功样本、失败样本、失败类型、单变量重试效果、是否应拆镜/补首帧/改资产/改提示词，以及是否建议回写编译器规则。未经用户明确确认满意，不得把候选结论写入正式正向训练库；但可以作为生产系统优化记录，标注为“生成后复盘候选”。

### 1. 锁定输入

提取并编号人物、关系、环境、时间、风格、对白、事件、产品和已有资产。区分用户事实与编译器假设。

只有以下情况暂停受影响镜头：人物身份矛盾、对白归属不明、必须绑定的产品或人物资产缺失、关键动作存在无法解决的因果矛盾。其余信息优先合理补全并标注。

修改既有项目脚本时，先定位本次改动影响的全部镜头编号、原镜头编号、文件名、资产 ID 和跨文件引用；再执行文本改动。新增成长阶段、回忆段、转场段或情绪承接段时，必须写清发生时间、人物年龄层/身份状态、场景时间段，以及它如何承接上一镜并推动下一镜。

### 2. 补全导演决策

确定已由用户确认的画幅、预计总时长、情绪曲线、镜头数量、每镜用途、景别、机位、构图、视线、动作、声音、切点和转场。若画幅未确认，暂停正式逐镜任务包，只输出画幅候选与确认问题。对编导新手同时解释“为什么这样拍”。

一个生成任务原则上只保留：一个稳定时空、一个主要说话者、一个主动作和一个摄影意图。复杂内容必须拆分。

摄影控制必须收敛：每个生成任务只写一个主要景别、一个机位和一个运镜。优先使用稳定组合，例如中景正面缓慢推进、特写静止、远景缓慢拉近或拉远。禁止在同一任务内写快速切换、多角度、多运镜或多个互相竞争的摄影意图。

关系戏必须明确人物距离、身体朝向和视线对象。亲情、照护、递接、牵手、床边陪护等镜头，不能只写“看着她”“温柔地说”，必须写清“谁靠近谁、距离多近、身体是否侧向对方、眼神落在哪里、说话对象是谁”。如果亲密关系镜头没有距离约束，Seedance 容易把母女拍成路人或让角色对空气说话。

每个分镜视频脚本必须包含可执行的时间信息和逻辑引导：

- 时间信息：年代/成长阶段、日夜或具体时段、季节或天气、角色年龄层；无法确定时用安全假设并标注。可复制提示词代码块内还必须写明建议生成总秒数，并用 `0-2秒 / 2-4秒 / 确认后` 等形式约束动作、对白、反应和尾帧。
- 逻辑引导：本镜承接上一镜的原因、角色为什么做这个动作、动作完成后为什么触发对白或反应、它如何把情绪推向下一镜。
- 生成引导：复杂动作必须用“先/随后/确认后/再”或逐秒时间轴写清，避免动作、对白和反应互相抢先。每个被写进提示词的动作都必须形成闭环：写清动作起始状态、执行方式、对象/道具的去向、完成后的最终可见状态，以及尾帧是否保持该状态；不得只写“拿走、挑出、递过去、推开、擦掉、放下”等过程而不说明物体最后在哪里、人物手停在哪里、画面如何验证动作已完成。
- 网页端提示词迁移结构：从已验证项目迁移到新项目时，先写一句“本镜核心目标”，说明本镜在全片中负责完成的钩子、压制、反转、揭示、承接或回扣；再写模型执行约束。不要把旧项目的题材、人物关系或情绪模式照搬到新项目，只迁移执行逻辑：先让模型知道这一镜为什么存在，再让模型按逐秒时间线执行。

### 3. 建立生产圣经

建立角色、场景、产品、声音和连续性记录。读取 `references/asset-keyframe-rules.md` 和 `references/voice-continuity-rules.md`，列出已有资产、必须补充资产和每镜引用用途。必须先根据完整视频脚本逐镜反推资产需求，再输出全局资产；不要只凭人物表或场景表泛化列资产。对每项缺失资产输出 GPT-IMAGE2 图片生成提示词、禁止项和验收标准；对每个需要锁定起点的镜头输出 GPT-IMAGE2 生产型首帧提示词，必要时输出尾帧提示词。

生成完整视频分镜、逐镜任务包或项目生产文件时，必须同步生成 GPT-IMAGE2 资产提示词，不能只输出 Seedance 视频提示词。GPT-IMAGE2 资产提示词至少覆盖：角色资产包、产品标准图、核心场景底图、关键道具、画面内文字/票据/照片道具、A级高风险生产型首帧、必要尾帧和G3确认门。角色/产品资产包用于筛选稳定参考图，不直接作为 Seedance 视频首帧。正式进入 Seedance 前，仍需为产品标准露出、精确文字、严格站位、跨镜连续性和必须锁定手部/道具起点的镜头制作16:9生产型首帧；低运动关系戏、静态手部留白、无精确产品/文字的探索镜头，可按无首帧测试分级规则先无首帧生成或测试一版。

先设计核心资产：主角、产品、核心场景和关键首帧。随后推荐角色正脸、产品或文字特写、最复杂动作三个高风险测试镜头，并说明测试收益和跳过测试的风险；测试是否执行由用户决定，不作为正式逐镜生成的硬阻塞。若完整脚本包含手机屏、便签、包装文字等画面内物件精确文字，应推荐文字直出测试；用户跳过测试时，仍可直接按逐镜操作单生成。默认每个 Seedance 视频不出现字幕：片名、结尾文案、对白字幕、旁白字幕、黑屏字卡都不得写入视频画面生成提示词。默认每个 Seedance 视频不生成背景音乐或背景声铺底：发布所需字幕、背景音乐和背景声音由后期添加。人物说话声音和场景中正在发生的必要事件声必须保留，并写成“剧情内声音”，不要写成背景音乐或氛围铺底。若用户明确要求人物声音在视频内生成，则只写必要人物对白和说话者规则，不写背景音乐；失败时优先重试、拆分镜头、降低运动、使用近景/首帧或上传音频参考。

注意：这里的“用户跳过测试时，仍可直接按逐镜操作单生成”只在核心资产硬门槛满足后成立。若高优先核心资产仍缺失，下一步必须先补资产或让用户确认降级，不得提前进入正式生成或生成登记。

当前生产优化：核心资产确认后，默认进入“网页端手动生成视频”模式。必须输出从 `SHOT_00` 或项目起始镜头到最后一镜的全镜头网页端操作单。每个镜头直接给出需要上传的图片资产、首帧/尾帧路径或无首帧理由、推荐生成方式、模型、画幅、时长、分辨率、运动强度、是否输入对白、剪辑手柄、完整可复制提示词、验收标准和失败后只改一个变量的建议。若用户已生成并满意但不发视频复盘，只根据已确认结果继续给下一镜生产说明。

### 4. 设计编辑镜头

先写面向人的导演分镜，再拆成 Seedance 任务。每个镜头必须回答：推进什么信息、制造什么情绪、为什么在这里切、如何连接前后镜头。

每个用于后期拼接的视频任务必须设计“剪辑手柄”，但不得为了长停留浪费 Seedance 生成时长。开头通常保留 0.2-0.3 秒稳定起始状态；结尾通常保留 0.3-0.4 秒稳定尾帧状态；长对白、合并镜头或情绪承接镜头可保留 0.5 秒稳定尾帧。稳定状态指人物站位、手部、道具、视线和情绪已落稳；尾部稳定段不得新增关键动作、不得新增台词，人物嘴部自然闭合或只保留无声呼吸。纯产品、便签、手机文字、片尾留白、手部静态特写等插入镜头只保留 0.3-0.4 秒可裁切尾帧；如果成片需要更长停留，优先在剪辑中使用定格帧、图片层或延长静帧，而不是消耗视频生成积分。需要无缝动作衔接的中段动作镜头可以缩短到 0.2-0.3 秒，但必须在任务里显式说明“用于动作接剪”。不要把整段时长全部塞满剧情动作。

### 5. 分配多模态输入

读取 `references/capability-matrix.md`。只分配真正影响身份、动作、声音或构图的图片、音频和视频参考，避免把全部素材塞进每个任务。

### 6. 编译逐镜任务

按照 `references/task-schema.md` 输出结构化任务和一段可复制的中文提示词。对白逐字保留，说明说话者、对象、语气、节奏、停顿、口型和非说话者嘴部状态。

编写可复制提示词时，按以下单镜头骨架组织内容，不得只写抽象风格词：

- 网页端代码块优先顺序：`生成规格/时长/画幅` → `本镜核心目标` → `场景时间` → `参考资产` → `角色锁定` → `镜头控制` → `声音白名单或无对白规则` → `逐秒时间线` → `剪辑手柄和尾帧` → `禁止项`。重要失败控制项可以前置，但逐秒时间线必须在代码块内清楚出现，推荐使用 `0-2秒...；2-3.5秒...；3.5-...秒...` 的连续分号结构，避免只写在代码块外部说明中。
- 角色锁定：年龄、性别、外貌、服装、身份、固定道具或动作锚点、必要对白声音特征；跨镜角色必须明确保持不变。
- 场景时间：地点、年代/成长阶段、日夜或具体时段、天气/季节、主要光源和空间锚点。
- 行为动作：角色起始状态、执行过程、细节动作、道具去向、完成后的最终可见状态。
- 镜头控制：一个景别、一个机位、一个运镜；说明构图和视线，不写多机位快速切换。
- 音频同步：需要 Seedance 直出人物对白时写说话者、对象、逐字台词、开口触发条件、语气、音色、语速、停顿、非说话者嘴部状态；场景中正在发生且应当发声的动作必须写成剧情内声音，例如脚步、门响、手机语音、碗筷声、雨打伞面。不得要求生成背景音乐、歌词、字幕、背景声铺底或解释性旁白字幕。
- 风格情绪：把风格和情绪翻译成可见的光线、色彩、距离、动作强度、表情和声音节奏；“电影感”“高级感”“压抑”等词只能作为总结，不得单独承担生成要求。
- 节奏结构：说明开头如何抓住注意力，中段如何递进，结尾如何落到反转、升华或留白；必须落到逐秒时间轴。

说话镜头必须绑定“说给谁听”。凡是画面中出现可见说话者，提示词必须写清说话者的视线对象、身体朝向和对方已停稳/已看见/已接住/已坐稳等触发条件。禁止只写“妈妈说”“女儿说”。如果角色说话时不该看镜头，必须写“不要看镜头，不把摄像机当成说话对象”。如果需要对手机语音说话，必须写清“看着手机屏幕/对手机中的聊天对象说话”，而不是“对着镜头说话”。

对话内容必须使用白名单：写清“仅允许以下台词，不得增加、重复、改写或提前说出其他台词”。多轮对白、两角色交替对白、音色容易混淆或模型出现额外台词时，下一版优先缩短时长、减少对白或拆镜，不要只在原长镜头里追加禁止项。

手机、便签、包装等画面内文字必须先决定观看对象。给观众读字时，使用稳定正面近景或插入镜头；给角色阅读或发送语音时，优先使用角色第一人称、过肩、手持朝向角色的视角，必要时再短暂转为可读近景。不得让人物长期对着摄像机/观众说手机语音，除非剧情明确是自拍视频。

对白和反应必须绑定清楚的触发条件。凡是同镜包含“人物入场/靠近/递接/握手/看见/低头看物/转头反应 + 对白”的任务，提示词必须写明动作完成后才开口：人物站位、视线对象、停顿、谁先说、谁后说、非说话者嘴部状态。禁止只用自然语言并列描述“跑来后停住、妈妈说、女儿反应”这类弱顺序；必须写成“女儿跑到妈妈面前停住并与妈妈对视后，妈妈才说……”。涉及可移动道具或小物件时，还必须写清道具落点和完成态，例如“夹出的葱花放到碗边小白瓷碟里，碗里不再有明显葱花，筷子停在碗沿旁”。若一个 8-15 秒合并镜头内有两个以上触发点，必须用逐秒时间轴或“先/随后/确认后/再”明确执行顺序，并在禁止项里加入“禁止提前开口、禁止对空处说话、禁止两人同时开口”。

可移动道具必须写清所有权和方向。递伞、递产品、推碗、递手机、擦手、握手、拉袖口等动作，必须写清道具从谁手里开始、朝哪个方向移动、谁接住、谁松手、最终停在哪里、尾帧由谁持有。产品信物镜头尤其要避免方向反转，例如“妈妈把黄精果递给女儿，女儿双手接稳后妈妈松手，尾帧黄精果在女儿手中”。

可复制提示词必须把剪辑手柄写成生成要求，而不是只放在验收说明里。常用句式：`开头保留0.2-0.3秒稳定起始状态；结尾保留0.3-0.4秒稳定尾帧，尾部稳定段不新增动作或对白，人物嘴部自然闭合，方便后期剪辑拼接。` 长对白或合并镜头可写成 `结尾保留0.5秒稳定尾帧`；产品、文字、片尾留白和手部静态特写不得默认写 0.8-1.0 秒视频尾帧，长停留交给后期定格或图片层。

每个镜头必须自包含：不得使用“同上一镜”“沿用前文”“其他不变”“略”等依赖上下文的省略写法。连续性要求可以引用前镜状态，但必须在本镜中重新写清需要继承的具体内容。

每次改动镜头编号或插入新镜头后，必须执行项目级对应关系检查：后续 `MERGE/SHOT/POST/INSERT` 编号是否唯一，备用插入引用是否指向新编号，叠化/回扣/回退说明是否仍正确，项目状态或建议表中的任务总数是否同步更新。检查完成后向用户说明已同步的文件和未同步的文件。

如果用户没有要求 JSON，仍然输出完整的人类可读任务包；如果输出 JSON，则它与人类可读版本必须使用相同的镜头 ID、台词、时长和资产 ID。

### 7. 检查和降级

读取 `references/quality-gates.md` 和 `references/retry-strategy.md`。有 JSON 时优先运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/validate_task_pack.ps1 <任务包.json>
```

具备 Python 环境时也可以运行 `scripts/validate_task_pack.py`。先降低动作、运镜或同镜信息量，再考虑改动内容。

用户发回生成视频时，必须先抽帧复核可见画面；若视频含可见口部对白，还应优先按项目内 `00-系统总控/音画同步检测流程_v01.md` 调用 `tools/verify_av_sync.py` 生成基础或 speech 模式报告。工具报告不替代用户听看确认，但必须用于发现音轨缺失、静音、明显错位或需要人工复核的口型风险。

## 镜头合并规则

Seedance 2.0 可生成 4-15 秒片段，并支持文本、图片、音频、视频多模态参考。不要机械地把所有动作拆成 5 秒小镜头；应根据网页端能力、故事清晰度和失败成本决定合并。

优先合并：

- 同一时间、同一地点、同一组人物、同一情绪段落。
- 一个镜头内只有一个主动作，外加一个自然延伸动作。
- 总时长可控制在 8-15 秒。
- 只有 1-2 句短对白，且不是两人复杂抢话。
- 产品只作为情绪物件，不要求商业级包装标准露出。
- 手部动作不是严格验收点，或可接受手部轻微不精确。
- 失败后可以在剪辑中切掉前后冗余，不会破坏因果。

必须拆分：

- 时间、地点、年代、人物年龄层发生变化。
- 精确中文、手机界面、便签、包装文字需要可读。
- 产品承担标准露出，必须保持包装正面、品名或品牌。
- 一个镜头内同时要求复杂手部、可见口型、情绪反应和产品准确。
- 两个以上人物轮流说话，且都清晰可见口型。
- 同一镜头内同时包含小物件精细动作、手机/便签/包装文字、道具递接方向和多句对白；若已经失败两次，不继续堆提示词，直接拆成动作镜头、文字/手机镜头和情绪对白镜头。
- 人物入场、递接、握手、拉袖口、推物等动作必须先完成，之后才触发关键对白或情绪反应，但网页端时长不足以写清逐秒顺序。
- 需要叠化、蒙太奇、跨时空对照或精确片内文字位置。
- 合并后超过 15 秒，或网页端当前只稳定支持 5 秒。

交付时必须给出两个版本：

1. `保守拆分版`：更稳，失败成本低，适合首轮生产。
2. `合并生产版`：利用 Seedance 2.0 能力减少镜头数，适合用户追求效率时直接生成。

如果用户要求“更少镜头”，优先生成 `13-镜头合并拆分建议_v01.md`，列出可合并、建议保留拆分、可后期处理的镜头，并说明理由。
